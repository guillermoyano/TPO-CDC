Utilizar etiquetas semánticas para estructurar el sitio. Se han utilizado etiquetas semánticas
  • Se hizo un Sidebar left, por lo cuál no agregamos la etiqueta Nav
  • Section. En cada página, encerrando el contenido general
  • Article. En el index, donde se separa cada una de las noticias
  • Se decidió no usar Footer para armonía visual

Contener un formulario de contacto con validación realizada mediante Javascript. Opcional: el formulario podrá enviar un email utilizando algún servicio externo destinado para ello. En "formulario.html", se realizó lo solicitado. Tiene la validación JS.

Utilizar al menos un Iframe, íconos de FontAwesome o Flaticon; y fuentes locales o bien de Google Fonts o similar Se han utilizado etiquetas semánticas
  • Se insertaron videos de YouTube a través de Iframe en "noticia-1(2 y 3).html"
  • En la página "nosotros.html" se agregaron íconos de LinkedIn y de Github
  • En distintas páginas se han insertado fuentes desde Google Fonts

Tener al menos una página totalmente responsive con 3 puntos de corte para 3 tamaños de dispositivos distintos. Si bien todas las páginas son responsivas, los puntos de corte se realizaron en "noticia-1(2 y 3).html"

Incorporar al menos una animación, transformación o una transición (se deberá indicar en qué parte del sitio web fue incorporada la animación). La animación se incorporó en todas las páginas, en la esquina superior izquierda. El logo se hizo giratorio en su propio eje

Poseer una estructura HTML maquetada con Flexbox y/o Grid Hemos utilizado ambas estrucutras

Consumir una API Rest desde JavaScript. Deberá informar la API utilizada al momento de entregar el TPO. Se agregó en "index.html" la api. Relacionada con el sitio, es de noticias.
